# Waypoint Navigation: Record & Replay

**Date:** 2026-04-18  
**Status:** Approved

## Overview

A teach-and-replay system for the Yahboom car in the main building. The operator drives the car to each location, records its AMCL pose, then runs a patrol node that navigates the car to those waypoints in order using Nav2.

## Architecture

Two nodes in `warehouse_vision`, one shared YAML file:

```
warehouse_vision/
  waypoint_recorder.py   ← subscribes /amcl_pose, saves pose on Enter keypress
  patrol_node.py         ← modified to load waypoints from YAML instead of hardcode
  waypoints.yaml         ← persisted waypoint list (generated by recorder)
```

## Component 1: Waypoint Recorder (`waypoint_recorder.py`)

- ROS2 node that subscribes to `/amcl_pose` (geometry_msgs/PoseWithCovarianceStamped)
- Background stdin thread: press **Enter** to save current pose, **Ctrl+C** to exit
- On startup, warns if no `/amcl_pose` received within 5 seconds (prompt user to set 2D Pose Estimate in RViz)
- Appends each saved pose as `{x, y, oz, ow}` to `~/waypoints.yaml`
- Prints confirmation: `Saved waypoint #N at (x=..., y=...)`

## Component 2: Patrol Node (`patrol_node.py`) — Modified

- Loads waypoints from `~/waypoints.yaml` at startup
- YAML file path configurable via ROS2 parameter `waypoints_file` (default: `~/waypoints.yaml`)
- All existing navigation logic unchanged: `followWaypoints`, progress logging, result reporting

## Workflow

1. `ros2 launch yahboomcar_nav navigation_dwb_launch.py`
2. Open RViz → click **2D Pose Estimate** → click+drag car's actual position on map
3. Confirm laser scan aligns with map walls
4. `ros2 run warehouse_vision waypoint_recorder` — drive to each spot, press Enter
5. `ros2 run warehouse_vision patrol_node` — car follows all saved waypoints

## YAML Format (`waypoints.yaml`)

```yaml
waypoints:
  - {x: 1.23, y: 0.45, oz: 0.0, ow: 1.0}
  - {x: 2.10, y: 1.30, oz: -0.52, ow: 0.85}
```

## Error Handling

- Recorder warns on startup if no `/amcl_pose` received within 5s
- Patrol node exits with clear error if `waypoints.yaml` is missing or empty
